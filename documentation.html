Installation Guide

Comprehensive steps to deploy your 100% offline outreach engine.

Hardware Guardrail: Ensure you have at least 8GB RAM. The 4-bit NF4 quantized engine requires ~4.5GB total runtime memory.

Option 1: Manual Installation

Recommended for developers wanting deep control over the environment.

1 Clone & Navigate

git clone https://github.com/rahulkate173/Outreach-Engine-.git cd Outreach-Engine-

2 Environment Setup

# Create virtual environment python -m venv venv # Activate (Windows) venv\Scripts\activate # Activate (macOS/Linux) source venv/bin/activate

3 Dependencies & Folders

pip install -r requirements.txt # Create required model and memory directories mkdir -p models_cache memory

4 Configuration

Link your MongoDB Cluster1 URI in the environment file.

cp .env.example .env # Edit .env with your MONGO_URI and settings

Option 2: Docker Installation

Deploy a containerized version of the FastAPI backend.

# Build and start services docker-compose up -d # Monitor logs docker-compose logs -f fastapi

Option 3: EXE (Windows)

# Build executable pyinstaller --onefile cli.py # Run from dist folder dist/cli.exe auth start

Quick Start: Using CLI

Interact with the system via our built-in command-line interface.

# Start with authentication flow python cli.py auth # Start server directly python cli.py server # Reset local model cache python cli.py reset-model

Quick Start: Direct Python

# Start FastAPI directly via Uvicorn python -m uvicorn app.main:app --reload # Server running on: http://localhost:8000
